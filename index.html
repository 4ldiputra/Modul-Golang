9. Komposisi (Composition)

9.1 Pengertian Komposisi

Apa Itu Komposisi?

Komposisi adalah cara di Go untuk membuat struct yang "mewarisi" field dari struct lain. Go tidak memiliki inheritance seperti OOP tradisional, melainkan menggunakan embedding (penyematan) struct.

Keuntungan Komposisi:

  • Lebih fleksibel - bisa menggabungkan beberapa struct
  • Menghindari kompleksitas hierarchy class
  • Reusability code yang lebih baik
  • Kode lebih mudah dibaca dan dipahami

9.2 Struct Embedding Sederhana

Contoh Dasar Embedding

package main import "fmt" // Struct dasar type Person struct { Name string Age int } // Struct Employee "mewarisi" Person dengan embedding type Employee struct { Person // Embedding (anonymous field) EmployeeID string Department string } func main() { // Membuat employee emp := Employee{ Person: Person{ Name: "Budi", Age: 25, }, EmployeeID: "EMP001", Department: "IT", } // Akses field Person secara langsung fmt.Println("=== DATA KARYAWAN ===") fmt.Println("Nama :", emp.Name) // bisa langsung! fmt.Println("Umur :", emp.Age) // tidak perlu emp.Person.Age fmt.Println("ID :", emp.EmployeeID) fmt.Println("Department:", emp.Department) // Atau akses melalui Person secara eksplisit fmt.Println("\n=== AKSES EKSPLISIT ===") fmt.Println("Nama:", emp.Person.Name) fmt.Println("Umur:", emp.Person.Age) }

Penjelasan:

  • Person di dalam Employee adalah anonymous field (field tanpa nama)
  • Field dari Person dapat diakses langsung dari Employee
  • Ini disebut "promoted fields" (field yang dipromosikan)
  • Bisa juga akses dengan emp.Person.Name jika perlu eksplisit

9.3 Komposisi dengan Multiple Struct

Sebuah struct bisa meng-embed beberapa struct sekaligus!

package main import "fmt" // Struct Address type Address struct { Street string City string ZipCode string } // Struct Contact type Contact struct { Email string Phone string } // Struct Person dengan multiple embedding type Person struct { Name string Age int Address // Embedding Address Contact // Embedding Contact } func main() { person := Person{ Name: "Ani", Age: 30, Address: Address{ Street: "Jl. Merdeka No. 10", City: "Jakarta", ZipCode: "12345", }, Contact: Contact{ Email: "ani@email.com", Phone: "081234567890", }, } // Akses field dari semua embedded struct fmt.Println("=== DATA LENGKAP ===") fmt.Println("Nama :", person.Name) // dari Person fmt.Println("Umur :", person.Age) // dari Person fmt.Println("Alamat :", person.Street) // dari Address fmt.Println("Kota :", person.City) // dari Address fmt.Println("Kode Pos:", person.ZipCode) // dari Address fmt.Println("Email :", person.Email) // dari Contact fmt.Println("Telepon :", person.Phone) // dari Contact // Akses secara eksplisit juga bisa fmt.Println("\n=== AKSES EKSPLISIT ===") fmt.Println("Kota:", person.Address.City) fmt.Println("Email:", person.Contact.Email) }

üí° Poin Penting tentang Komposisi

  • ‚úÖ Embedding: Cara Go untuk "mewarisi" field tanpa inheritance
  • ‚úÖ Promoted Fields: Field embedded struct bisa diakses langsung
  • ‚úÖ Anonymous Field: Field tanpa nama (hanya tipe struct-nya)
  • ‚úÖ Multiple Embedding: Bisa embed lebih dari satu struct
  • ‚úÖ Akses Eksplisit: Bisa akses dengan structName.EmbeddedStruct.Field
  • ‚ö†Ô∏è Name Conflict: Hati-hati jika ada field dengan nama sama dari multiple embedding
  • üí° Lebih Simple: Komposisi lebih mudah dipahami dibanding inheritance

üîÑ Komposisi vs Inheritance

Aspek Inheritance (OOP Tradisional) Composition (Go)
Hubungan "is-a" (Mobil adalah Kendaraan) "has-a" (Mobil memiliki data Kendaraan)
Fleksibilitas Kaku, hierarchy tetap Fleksibel, mudah diubah
Kompleksitas Bisa sangat kompleks Lebih sederhana
Kemudahan Perlu pemahaman OOP Lebih mudah dipahami
Reusability Terbatas pada hierarchy Lebih tinggi dan fleksibel

üìù Kapan Menggunakan Komposisi?

Gunakan komposisi ketika:

  • ‚úì Ingin menggabungkan beberapa tipe data berbeda
  • ‚úì Perlu reuse field dari struct lain
  • ‚úì Ingin struktur data yang fleksibel
  • ‚úì Membuat sistem yang kompleks tapi tetap mudah dipahami

Contoh kasus yang cocok:

  • Sistem manajemen (karyawan, mahasiswa, dll)
  • E-commerce (berbagai jenis produk)
  • Perpustakaan (buku, majalah, DVD)
  • Rental (mobil, motor, sepeda)
  • Hotel (berbagai tipe kamar)
